# Events

## Вступ

Events (події) - це механізм, за допомогою якого JavaScript реагує на дії користувача або зміни в браузері. Для Senior QA Automation Engineer розуміння Events критично важливе, оскільки всі взаємодії користувача (кліки, введення тексту, натискання клавіш) генерують події, які потрібно правильно обробляти в тестах. Вміння працювати з Events дозволяє симулювати дії користувача, перевіряти обробку подій та налагоджувати проблеми з взаємодією.

## Теоретичне пояснення

Події в браузері виникають при різних діях: клік миші, натискання клавіші, зміна значення поля, завантаження сторінки тощо. JavaScript може слухати ці події через event listeners та реагувати на них. Події поширюються через DOM дерево в двох фазах: capturing (вниз) та bubbling (вгору). Розуміння цього механізму дозволяє правильно обробляти події та запобігати небажаної поведінки.

## Основні концепції

### Типи подій

- **click**: Клік миші
- **change**: Зміна значення (для input, select)
- **focus/blur**: Фокус на/з елемента
- **input**: Введення тексту
- **submit**: Відправка форми
- **keydown/keyup**: Натискання клавіш

### Структура об'єкту Event

- **type**: Тип події
- **target**: Елемент, на якому виникла подія
- **currentTarget**: Елемент, на якому встановлений listener
- **preventDefault()**: Запобігання стандартної поведінки
- **stopPropagation()**: Зупинка поширення події

### Event Listeners

- **addEventListener()**: Додавання listener
- **removeEventListener()**: Видалення listener
- **onclick**: Inline обробник (не рекомендується)

### Event Bubbling та Capturing

- **Bubbling**: Подія поширюється вгору по DOM дереву
- **Capturing**: Подія поширюється вниз по DOM дереву
- **stopPropagation()**: Зупинка поширення

## Поради для Senior QA Engineer

1. **Використовуйте правильні події**: click для кнопок, change для select
2. **Чекайте на події**: Використовуйте wait для асинхронних подій
3. **Перевіряйте preventDefault**: Перевіряйте, чи правильно запобігається стандартна поведінка
4. **Тестуйте bubbling**: Розумійте, як події поширюються
