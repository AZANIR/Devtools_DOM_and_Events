# DevTools - Elements

## Вступ

Панель Elements в Chrome DevTools дозволяє інспектувати та редагувати HTML та CSS в реальному часі. Для Senior QA Automation Engineer Elements панель є основним інструментом для пошуку правильних селекторів, аналізу структури DOM, перевірки атрибутів елементів та розуміння ієрархії елементів. Вміння ефективно використовувати Elements панель дозволяє швидко знаходити унікальні селектори, розуміти структуру сторінки та підготовлювати надійні тести.

## Теоретичне пояснення

Панель Elements відображає повну структуру DOM дерева поточної сторінки. Вона дозволяє вибирати елементи на сторінці (через кнопку "Select element" або через код), переглядати їх HTML, CSS стилі, обчислені стилі, event listeners та інші властивості. Можна редагувати HTML та CSS в реальному часі, що дозволяє тестувати різні сценарії без зміни вихідного коду.

Для автоматизованого тестування Elements панель дозволяє знаходити найкращі селектори для елементів (ID, class, data-атрибути, XPath), розуміти структуру DOM для написання правильних селекторів, перевіряти атрибути елементів для валідації, та аналізувати ієрархію елементів для навігації по DOM дереву. Це особливо важливо при роботі зі складними SPA, де DOM структура може динамічно змінюватися.

## Основні концепції

### Пошук селекторів

- **CSS селектори**: Найпростіший та найшвидший спосіб
- **XPath**: Потужний, але складніший у підтримці
- **Data-атрибути**: Найкращий вибір для тестування (data-testid)
- **Комбіновані селектори**: Комбінація різних типів

### Аналіз структури DOM

- **Ієрархія елементів**: Розуміння parent-child відношень
- **Sibling елементи**: Елементи на одному рівні
- **Nested структури**: Вкладені елементи

### Перевірка атрибутів елементів

- **HTML атрибути**: id, class, data-*, aria-*
- **Custom атрибути**: data-testid, data-cy
- **Властивості**: textContent, innerHTML

## Поради для Senior QA Engineer

1. **Використовуйте data-testid**: Найкращий вибір для стабільних селекторів
2. **Уникайте залежності від CSS**: CSS класи можуть змінюватися
3. **Перевіряйте унікальність**: Завжди перевіряйте, що селектор знаходить тільки один елемент
4. **Використовуйте Copy selector**: Функція DevTools для копіювання селектора
