# DevTools - Console

## Вступ

Панель Console в Chrome DevTools є потужним інструментом для виконання JavaScript коду, налагодження та тестування веб-додатків. Для Senior QA Automation Engineer Console є незамінним інструментом для швидкого тестування селекторів, перевірки DOM елементів, виконання JavaScript команд та налагодження проблем. Вміння ефективно використовувати Console дозволяє швидко знаходити проблеми, тестувати гіпотези та підготовлювати код для автоматизованих тестів.

**Важливо:** Всі приклади в цій документації працюють на реальному сайті:
- **example.com**: https://example.com - відкрийте цей сайт та виконайте приклади в Console

Всі приклади можна виконати одразу в DevTools Console або в Cypress тестах.

## Теоретичне пояснення

Панель Console в Chrome DevTools надає REPL (Read-Eval-Print Loop) середовище для виконання JavaScript коду безпосередньо в контексті поточної сторінки. Це дозволяє отримувати доступ до всіх глобальних змінних, функцій та об'єктів сторінки, виконувати DOM маніпуляції, тестувати селектори та перевіряти поведінку додатку в реальному часі.

Для автоматизованого тестування Console дозволяє швидко перевіряти селектори перед написанням тестів, тестувати JavaScript код перед інтеграцією в тести, налагоджувати проблеми шляхом виконання команд вручну, та перевіряти стан додатку через доступ до глобальних змінних. Це особливо корисно при роботі зі складними селекторами, динамічним контентом та асинхронними операціями.

## Основні концепції

### Виконання JavaScript команд

- **Пряме виконання**: Введення коду та виконання Enter
- **Multi-line код**: Використання Shift+Enter для багаторядкового коду
- **Історія команд**: Використання стрілок вгору/вниз для навігації
- **Автодоповнення**: Tab для автодоповнення команд

### Вивід інформації для налагодження

- **console.log()**: Звичайний вивід інформації
- **console.error()**: Вивід помилок
- **console.warn()**: Вивід попереджень
- **console.table()**: Вивід у вигляді таблиці
- **console.group()**: Групування виводу

### Тестування селекторів та DOM елементів

- **querySelector()**: Пошук елементів за CSS селектором
- **querySelectorAll()**: Пошук всіх елементів
- **getElementById()**: Пошук за ID
- **Перевірка наявності**: Перевірка, чи елемент існує
- **Перевірка властивостей**: Перевірка текстів, атрибутів, стилів

## Поради для Senior QA Engineer

### Best Practices

1. **Використовуйте console.table()**: Для виводу масивів та об'єктів у зручному форматі
2. **Групуйте вивід**: Використовуйте console.group() для організації виводу
3. **Використовуйте строгі перевірки**: Завжди перевіряйте, чи елемент існує перед використанням
4. **Тестуйте селектори**: Завжди тестуйте селектори в Console перед використанням в тестах
5. **Зберігайте корисні команди**: Створюйте функції-помічники для часто використовуваних операцій

### Advanced техніки

1. **Breakpoints в коді**: Використовуйте `debugger;` для зупинки виконання
2. **Моніторинг подій**: Відстежуйте події через addEventListener
3. **Performance timing**: Використовуйте performance API для вимірювання часу
4. **Network monitoring**: Перехоплюйте fetch/XHR запити через Console
5. **Storage monitoring**: Відстежуйте зміни в LocalStorage/SessionStorage

### Налагодження

1. **Перевірка помилок**: Завжди перевіряйте Console на наявність помилок
2. **Використання stack trace**: Використовуйте console.trace() для відстеження викликів
3. **Умовний вивід**: Використовуйте console.assert() для перевірки умов
4. **Таймері**: Використовуйте console.time() та console.timeEnd() для вимірювання часу
