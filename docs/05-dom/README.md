# DOM (Document Object Model)

## Вступ

DOM (Document Object Model) - це програмний інтерфейс для HTML та XML документів, який представляє структуру документа як дерево об'єктів. Для Senior QA Automation Engineer розуміння DOM критично важливе, оскільки всі автоматизовані тести взаємодіють з DOM для знаходження елементів, читання даних та виконання дій. Вміння працювати з DOM дозволяє писати ефективні та надійні тести.

## Теоретичне пояснення

DOM представляє HTML документ як дерево вузлів, де кожен елемент, атрибут та текстовий вміст є вузлом. JavaScript може маніпулювати цим деревом: додавати, видаляти та модифікувати елементи. Для автоматизації тестування розуміння DOM дозволяє правильно знаходити елементи, перевіряти їх стан, маніпулювати ними та відстежувати зміни.

## Основні концепції

### Методи пошуку елементів

- **getElementById()**: Пошук за унікальним ID
- **querySelector()**: Пошук за CSS селектором (перший елемент)
- **querySelectorAll()**: Пошук всіх елементів за селектором
- **getElementsByClassName()**: Пошук за класом
- **getElementsByTagName()**: Пошук за тегом

### Маніпуляція елементами

- **Додавання**: createElement(), appendChild()
- **Видалення**: removeChild(), remove()
- **Модифікація**: textContent, innerHTML, setAttribute()

### Робота з атрибутами

- **getAttribute()**: Отримання значення атрибута
- **setAttribute()**: Встановлення атрибута
- **removeAttribute()**: Видалення атрибута
- **hasAttribute()**: Перевірка наявності

## Поради для Senior QA Engineer

1. **Використовуйте querySelector**: Найсучасніший та найгнучкіший метод
2. **Перевіряйте наявність**: Завжди перевіряйте, чи елемент існує перед використанням
3. **Уникайте innerHTML**: Використовуйте textContent для безпеки
4. **Розумійте ієрархію**: Розуміння parent-child відношень важливе для навігації
